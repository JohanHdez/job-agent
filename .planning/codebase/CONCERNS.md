# CONCERNS.md — Technical Debt, Issues & Areas of Concern

Generated by gsd-codebase-mapper (concerns focus).

## 1. Test Coverage — CRITICAL

**Impact:** High | **Effort:** High

Zero test suite across entire codebase. No test files exist in any package:
- `packages/cv-parser/` — no tests
- `packages/linkedin-mcp/` — no tests
- `packages/ats-apply/` — no tests
- `packages/job-search/` — no tests
- `packages/api/` — no tests
- `apps/cli/` — no tests

CLAUDE.md mandates ≥60% (web), ≥80% (shared), ≥70% (microservices) but none of these thresholds are currently met.

**Recommended fix:** Add Vitest for packages, Jest + Supertest for microservices. Start with shared utilities and logger.

---

## 2. LinkedIn Selector Fragility — HIGH

**Impact:** High | **Effort:** Medium

Bilingual selectors in `packages/linkedin-mcp/src/` are untested. LinkedIn frequently changes its DOM structure. Any layout change breaks the entire application pipeline with no fallback.

- No selector validation or health-check mechanism
- No retry with alternative selectors
- Selectors not centralized in a single constants file yet

**Recommended fix:** Move all selectors to `linkedin.constants.ts`. Add selector health-check smoke test. Log selector failures with context.

---

## 3. Single-Session In-Memory State — HIGH

**Impact:** High | **Effort:** Medium

No persistence layer for job application state between sessions. If the process crashes mid-run:
- Applied jobs list is lost
- No deduplication across sessions
- Risk of re-applying to the same jobs

**Recommended fix:** Persist `applications.json` atomically after each application. Load on startup to build seen-jobs set.

---

## 4. Missing Environment Variable Validation — HIGH

**Impact:** High | **Effort:** Low

No schema validation for `.env` at startup. Missing `ANTHROPIC_API_KEY` or `LINKEDIN_EMAIL` only surfaces as a runtime error deep in execution, not at startup.

**Recommended fix:** Add Zod or manual guard in `apps/cli/src/index.ts` to validate all required env vars before any work begins.

---

## 5. Rate Limiting Not Globally Enforced — HIGH

**Impact:** High | **Effort:** Low

CLAUDE.md mandates 3–5s delays between scroll pages and 8–12s between Easy Apply submissions. These are documented but not consistently enforced throughout the codebase. Manual `setTimeout` calls may be missing or inconsistent.

**Recommended fix:** Create a centralized `delay(min, max)` utility. Enforce via a rate-limiter wrapper around all LinkedIn interactions.

---

## 6. CV Parser Heuristic Fragility — MEDIUM

**Impact:** Medium | **Effort:** Medium

`packages/cv-parser/` uses regex-based PDF parsing. Complex or non-standard CV formats (multi-column, tables, graphics-heavy) may fail silently, producing incomplete `ProfessionalProfileType` objects.

**Recommended fix:** Add confidence scores to parsed fields. Fall back to Claude API for low-confidence parses. Validate output against schema before writing `output/profile.json`.

---

## 7. Unbounded Resource Consumption — MEDIUM

**Impact:** Medium | **Effort:** Low

Playwright browser instances may not be properly cleaned up on errors. Zombie browser processes can accumulate during long sessions or crashes.

**Recommended fix:** Wrap all Playwright operations in `try/finally` to ensure `browser.close()` is always called. Add process exit handlers.

---

## 8. Error Handling Inconsistency — MEDIUM

**Impact:** Medium | **Effort:** Medium

Error handling is inconsistent across packages. Some Playwright operations silently swallow errors, making debugging difficult. Some async functions lack typed `try/catch` blocks as mandated by CLAUDE.md.

**Recommended fix:** Audit all `catch` blocks. Ensure every catch logs the error with context via the structured logger. No silent error suppression.

---

## 9. No Credential Rotation — MEDIUM

**Impact:** Medium | **Effort:** Medium

LinkedIn session tokens/cookies are never refreshed. Sessions may expire mid-run without detection, causing silent failures that look like scraping errors.

**Recommended fix:** Detect session expiration (redirect to login page). Re-authenticate and retry. Alert user if re-auth fails.

---

## 10. Incomplete ATS Support — MEDIUM

**Impact:** Medium | **Effort:** High

`packages/ats-apply/` has gaps in handling custom Greenhouse/Lever fields (e.g., custom dropdowns, file uploads beyond CV, cover letter text areas). These fail silently or produce incomplete applications.

**Recommended fix:** Enumerate all known ATS field types. Implement handlers for each. Log unhandled fields as warnings.

---

## 11. Application Status Enum Misalignment — MEDIUM

**Impact:** Medium | **Effort:** Low

`ApplicationStatusType` in `packages/shared/types/` defines `'applied' | 'failed' | 'skipped' | 'already_applied'`. Some code paths may use string literals that don't match, bypassing type safety.

**Recommended fix:** Audit all status assignments. Use the enum type exclusively. Enable strict TypeScript checks.

---

## 12. MongoDB Connection Isolation Not Validated — MEDIUM

**Impact:** Medium | **Effort:** Low

CLAUDE.md mandates that each microservice uses its own isolated MongoDB database. This isolation is architectural intent but not validated at runtime — a misconfigured `MONGO_URI` could cause cross-service data access.

**Recommended fix:** Add startup validation that confirms each service connects to its designated database name. Log connection target on startup.

---

## 13. No Duplicate Application Detection Across Sessions — MEDIUM

**Impact:** Medium | **Effort:** Low

Without persistent state, re-running the CLI applies to the same jobs across sessions. This can result in duplicate applications, which may flag the LinkedIn account.

**Recommended fix:** Maintain a persistent `applied-job-ids.json` file. Check before each application attempt.

---

## 14. Microservice Authentication Not Implemented — LOW

**Impact:** Low (currently single-user) | **Effort:** High

Auth guards are defined in `apps/microservices/user-service/` but are not protecting all endpoints. The API gateway in `packages/api/` has no authentication middleware.

**Recommended fix:** Implement JWT validation middleware in the API gateway. Protect all non-public routes.

---

## 15. Pipeline Orchestration Timing Assumptions — LOW

**Impact:** Low | **Effort:** Low

The CLI orchestration in `apps/cli/src/index.ts` uses sequential awaits. If any step (cv-parser, job-search, ats-apply) hangs indefinitely, the pipeline stalls with no timeout or recovery.

**Recommended fix:** Add per-step timeouts using `Promise.race`. Log step durations. Alert if a step exceeds expected duration.

---

## 16. Report Generation Has No Recovery — LOW

**Impact:** Low | **Effort:** Low

If report generation fails (e.g., template error, missing data), the CLI exits without surfacing the error clearly. The user may not realize the report was not generated.

**Recommended fix:** Wrap report generation in explicit try/catch. Display clear error message if report fails. Continue execution (report is non-critical).
