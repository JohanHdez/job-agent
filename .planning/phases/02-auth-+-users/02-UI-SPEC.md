## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap within button (`gap: '4px'`), chip internal padding |
| sm | 8px | Chip group gap, inline element spacing, form field gap |
| md | 16px | Default element vertical spacing, section padding internal, card gap |
| lg | 24px | Card padding (`padding: '24px'`), section-to-section gap |
| xl | 32px | Page header bottom margin, major section breaks |
| 2xl | 48px | Empty state vertical padding (top + bottom) |
| 3xl | 64px | Empty state container padding (`padding: '64px 32px'`), page bottom safe area |

Exceptions:
- Dropzone internal padding: 48px 32px (established in `CvDropzone`) — keep as-is.
- Profile page max-width container: `padding: '32px 24px 64px'` — page chrome top/bottom padding.
- Touch targets: all interactive elements (buttons, pill toggles, chip remove buttons) must be minimum 36px in one dimension. Primary CTA buttons use 44px height minimum.

Source: Confirmed from `ProfilePage.tsx` (`SectionCard` padding 24px), `ConfigPage.tsx` (section padding 16px 24px), `CvDropzone` (48px 32px dropzone padding).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f0f14` | Page background, full-page containers |
| Secondary (30%) | `#1a1a24` | Cards, section panels, dropzone container, input surface |
| Input bg | `#12121a` | Text input fields, ChipInput background (darker inset) |
| Border | `#2a2a38` | Card borders, input borders, section dividers |
| Accent (10%) | `#6366f1` | See reserved list below |
| Text primary | `#f0f0f8` | Headings, names, high-emphasis labels |
| Text default | `#e2e2e8` | Body text, field values |
| Text muted | `#9090a8` | Subtitles, helper text, meta rows |
| Text faint | `#6b7280` | Captions, timestamps, legal copy |
| Destructive | `#ef4444` | Error banners, delete actions, validation errors |
| Success | `#22c55e` | Upload success, save confirmation banner |

**Accent `#6366f1` is reserved for:**
- Primary CTA button background (gradient: `linear-gradient(135deg, #6366f1, #7c3aed)`)
- Active/selected state for pill toggles and chip inputs
- Section header icon tint
- Upload icon container background (rgba opacity 10%)
- Spinner border-top for loading states
- Badge chip backgrounds (rgba 12% opacity with rgba border 25%)
- Logo mark gradient (`linear-gradient(135deg, #6366f1, #8b5cf6)`)
- Ambient background glow (`radial-gradient` from top, rgba 10–15%)
- `accentColor` on range input

**Accent is NOT used for:** plain text color, card backgrounds, borders in default state, destructive actions.

Source: Confirmed from `ConfigPage.tsx` (`C` token object), `LoginPage.tsx` (card shadow, logo gradient), `ProfilePage.tsx` (dropzone drag state, chip variant "indigo").

## Component Inventory (Phase 2 additions)

The following new components must be built for Phase 2 using the established patterns above. Each must match the visual language of the existing components in the codebase.

### New: `ProfileSetupPage.tsx`

**Route:** `/profile/setup`
**Purpose:** First-login onboarding gate. User fills critical profile fields before reaching dashboard.
**Layout:** Centered card, max-width 640px, same card pattern as `LoginPage` (`#1a1a24` surface, `1px solid #2a2a38` border, `border-radius: 24px`).
**Primary focal point:** The page heading "Complete Your Profile" — largest element on screen at 24px weight 700, draws the eye before the form fields.
**Sections:**
- Page title: "Complete Your Profile" (24px, weight 700, `#f0f0f8`, tracking -0.04em)
- Sub-copy: "Fill in the fields below so the agent can find the right jobs for you." (13px, `#6b7280`)
- Editable fields: fullName (text), headline (text), seniority (PillToggle group: Junior / Mid / Senior / Lead), skills (ChipInput), at least 1 experience entry (company + title text inputs, required).
- Primary CTA: "Save Profile" (full-width, indigo gradient button)
- Secondary: "Skip for now" (text-only link, 13px, `#6b7280`) — only shown if the user has some data already (graceful degradation; CONTEXT.md allows manual completion).

**Required fields indicator:** Fields missing from the completeness threshold (skills, seniority, experience) show a small red dot (`#ef4444`, 6px) or `(required)` label in 12px `#ef4444`.

### New: Incomplete Profile Banner (in `ProfilePage.tsx`)

**Trigger:** `checkProfileCompleteness()` returns missing fields (PROF-04).
**Appearance:** Full-width banner between page header and content area.
- Background: `rgba(245,158,11,0.08)` (amber — warning, not error)
- Border: `1px solid rgba(245,158,11,0.25)`
- Border-radius: 12px
- Padding: 14px 16px
- Icon: alert-triangle SVG, `#fbbf24`, 16x16
- Text: "Your profile is missing: {comma-separated field list}. Add them to improve job matching." — 13px, `#fbbf24`
- CTA: "Complete profile" inline link — 13px weight 700, `#6366f1`, underline on hover

### New: Preset Management Section (in `ConfigPage.tsx`)

**Location:** New section at the top of `ConfigPage`, before the Keywords section.
**Purpose:** Save/switch/delete named search presets (SRCH-01, SRCH-02). Max 5 presets.

**Sub-components needed:**

**Preset selector row:**
- Label: "Active Preset" (FieldLabel style)
- Dropdown/select element: lists all saved preset names + "Custom (unsaved)" option. Height 44px, same `baseInput` style.
- "Load" button: secondary style — `#1a1a24` background, `1px solid #2a2a38` border, 13px weight 700, `#e2e2e8` text, 36px height.

**Save as preset:**
- Text input for preset name (placeholder: "Name this preset…") — `baseInput` style, height 44px
- "Save Preset" button: indigo accent, 13px weight 700, height 36px, border-radius 8px. Disabled when 5 presets already exist.
- When at limit (5/5): show inline warning "Maximum 5 presets reached. Delete one to save a new preset." — 12px, `#fbbf24`.

**Preset list:**
- Renders each saved preset as a row: preset name (14px weight 400, `#e2e2e8`), "Activate" pill button (only when not active), "Delete" icon button.
- Active preset row has `1px solid rgba(99,102,241,0.4)` left border accent or subtle indigo background tint `rgba(99,102,241,0.06)`.
- Delete button: trash icon SVG 14x14, `#6b7280` default, `#ef4444` on hover. No inline confirmation — uses modal (see destructive actions below).

## Interaction States

### Auth Callback Page (migrated)

| State | Visual |
|-------|--------|
| Loading (exchanging code) | Existing spinner — 32px ring, `rgba(99,102,241,0.2)` border, `#6366f1` borderTopColor, 0.75s spin. Copy: "Completing sign-in…" (`#9090a8`, 14px). |
| Error (code expired / exchange failed) | Auto-redirect to `/login?error=auth_failed` — existing LoginPage error banner handles display. |
| Success | Redirect to `/profile/setup` (if no profile) or `/config` (if profile complete). |

### CV Upload (existing + extended for Phase 2)

| State | Visual |
|-------|--------|
| Idle | Dashed border `#2a2a38`, upload icon indigo container. Copy: "Drop your PDF here or click to upload". |
| Drag over | Border `#6366f1`, background `rgba(99,102,241,0.06)`. |
| Uploading | Spinner replaces icon. Border `#2a2a38`. Copy: "Uploading…". |
| Success | Border `#4ade80`. Status pill: green background, `#4ade80` text. |
| Error | Border `#f87171`. Status pill: red background, `#f87171` text. |
| Merge conflict (manual edits exist) | No visual conflict shown — merge is automatic (fill-empty-only per CONTEXT.md). Show success state. |

### Profile Edit Mode (new in Phase 2)

Phase 2 adds an edit mode toggle to `ProfilePage`. The page has two modes:

**View mode (default):** Existing read-only `ProfileDisplay` component.
**Edit mode:** Form fields replace display-only elements in-place within each `SectionCard`. Fields use `baseInput` style (`#12121a` background, `1px solid #2a2a38` border, 8px border-radius, 14px text).

Toggle trigger: "Edit Profile" button in page header. Same position as existing "Import CV" button, secondary style when in view mode, switches to "Discard Changes" (text link style, `#6b7280`) when in edit mode.
Save trigger: "Save Changes" primary CTA (indigo gradient, full-width below the last section). Show spinner during mutation. Show success banner on save.

## Accessibility Requirements

- All interactive elements must have `aria-label` when they lack visible text (icon-only buttons, upload dropzone — already present in existing code).
- Error banners must have `role="alert"` (established in `LoginPage.tsx` and `ConfigPage.tsx`).
- Form fields must have associated `<label>` elements or `aria-label`.
- Minimum contrast ratio 4.5:1 for body text against card background (`#e2e2e8` on `#1a1a24` passes).
- Spinner elements must have `aria-busy="true"` on the container during loading states.
- Required fields in `ProfileSetupPage` must have `aria-required="true"`.
- Delete confirmation inline pattern must trap focus on the confirmation row until resolved or auto-collapsed.
