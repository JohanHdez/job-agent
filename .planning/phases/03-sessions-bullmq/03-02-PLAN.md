Methods:

- `async createSession(userId: string, config: Record<string, unknown>): Promise<{ sessionId: string }>` — check for active session (status in ['queued', 'running'] for userId), throw `ConflictException({ code: 'SESSION_ALREADY_ACTIVE', sessionId: active._id.toHexString() })` if found. Create session doc with status='queued', config, userId. Enqueue BullMQ job: `this.searchSessionQueue.add('run-session', { sessionId: session._id.toHexString(), userId }, { attempts: 2, backoff: { type: 'exponential', delay: 5000 }, removeOnComplete: { count: 100 }, removeOnFail: { count: 100 } })`. Return `{ sessionId }`.

- `async findByIdForUser(sessionId: string, userId: string): Promise<SessionDocument>` — `this.sessionModel.findOne({ _id: sessionId, userId })`. Throw `NotFoundException('Session not found')` if null.

- `async cancelSession(sessionId: string, userId: string): Promise<SessionDocument>` — find by id+userId (throw 404 if missing). Set `status='cancelled'`, `completedAt=new Date()`. Save and return.

- `async appendEvent(sessionId: string, type: string, data: Record<string, unknown>): Promise<number>` — Two atomic operations: (1) `findByIdAndUpdate(sessionId, { $inc: { nextEventId: 1 } }, { new: true, projection: { nextEventId: 1 } })` to get the new id. (2) `updateOne({ _id: sessionId }, { $push: { events: { $each: [{ id: eventId, type, data, timestamp: new Date().toISOString() }], $slice: -100 } } })`. Return eventId.

- `subscribeToEvents(sessionId: string, handler: (event: StoredSessionEvent) => void, close$: Observable<void>): Promise<void>` — Subscribe to Redis channel `session:${sessionId}:events`. Wire `this.redisSubscriber.on('message', onMessage)` where onMessage parses JSON and calls handler. Use `close$.pipe(take(1)).subscribe(() => { this.redisSubscriber.removeListener('message', onMessage); this.redisSubscriber.unsubscribe(channel); })` for cleanup.

Add JSDoc on every public method. Use structured logger `createLogger('SessionsService')` from `@job-agent/logger`.
